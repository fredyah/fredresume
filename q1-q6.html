About Me

自我介紹

各位面試官好，我叫葉紹棠，也可以叫我 Fred。

我目前的職涯方向主要是 DevOps、AI 系統工程師，以及雲端與系統維運開發。過去我的工作經驗比較跨領域， 早期有業務、設備維修與客戶溝通的經驗，後來逐步轉向系統工程、DevOps、自動化維運，以及近年的 AI 系統整合。 這樣的背景讓我不只具備技術實作能力，也比較能理解需求、溝通問題，並把技術方案真正落地。

在技術方面，我最熟悉的是 Linux 系統維運與自動化處理，包含 Ubuntu、CentOS、NGINX、crontab、 Shell Script、Python Script、服務部署、Log 分析與問題排查。過去工作中，我常需要處理伺服器環境建置、 服務異常排查、備份還原、排程任務，以及監控告警整合。

在 DevOps 方面，我有實際使用 GitLab、GitLab Runner、CI/CD Pipeline、Docker、Dockerfile、 Kubernetes、Rancher、Helm、Argo CD 等工具的經驗。工作上會協助專案從程式建置、Image 打包、 部署到 Kubernetes 環境，以及後續的維運監控。監控方面，我也有接觸 Prometheus、Grafana、Zabbix、Graylog， 並且會透過 Line、Telegram 或 Discord Bot 做告警通知，讓問題能更快被發現與處理。

近年我也投入很多在 AI 系統與應用平台建置。例如我有建置與使用 Ollama、vLLM、OpenWebUI、 AnythingLLM、n8n、RAG、Milvus、Hugging Face 模型 等經驗，也參與過 AI 相關後端 API 的開發， 包含 Python API、C# Middleware API，以及文件分塊、metadata、向量資料庫串接等流程。 對我來說，AI 不只是模型本身，更重要的是如何把模型、資料、API、容器平台和實際應用場景整合起來。

資料庫方面，我有使用 MySQL、PostgreSQL、Redis、SQLite、Milvus、Qdrant 等經驗， 能處理基本查詢、備份還原、權限設定與服務串接。雲端方面則接觸過 AWS、Azure、GCP、Aliyun、Tencent Cloud， 其中 AWS 使用經驗包含 EC2、VPC、Security Group、ALB、RDS、EKS、Route53、CloudWatch 等服務。

我認為自己的優勢是：第一，具備系統維運、DevOps、AI 平台與後端 API 的整合能力；第二，因為過去有業務與現場服務經驗， 所以在跨部門溝通、需求釐清與問題協調上比較有彈性；第三，我很重視自動化與可維護性，面對重複性任務時， 會傾向用腳本、流程或監控機制來降低人工錯誤。

未來我希望能持續往 DevOps / SRE / AI 系統整合 的方向發展，強化 Kubernetes、IaC、MLOps、 監控告警與高可用架構設計能力。也希望能在團隊中協助提升系統穩定性、部署效率與 AI 應用落地能力。

以上是我的自我介紹，謝謝。
